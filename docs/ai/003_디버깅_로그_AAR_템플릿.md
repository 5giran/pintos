# 디버깅 로그 AAR 템플릿

이 템플릿은 Pintos 작업 중 테스트 실패, 커널 패닉, 회귀, 빌드 실패가 발생했을 때 팀원이 같은 방식으로 기록하기 위한 문서다.

목표는 "무슨 문제가 있었고, 어디서 발생했고, 어떤 커밋에서 어떻게 해결했는지"를 나중에 다시 찾을 수 있게 남기는 것이다. 구현 코드를 길게 붙이기보다 재현 조건, 실패 로그, 원인 판단, 수정 커밋, 검증 결과를 우선 기록한다.

## 제목

```text
[AAR] <테스트명 또는 증상> 실패 문제 해결
```

예시:

```text
[AAR] create-null 커널 패닉 문제 해결
[AAR] exec-once 실패 문제 해결
[AAR] setup_stack false 반환 문제 해결
```

## 2. 문제 상황

### 2.1. 기대한 동작

- 어떤 기능이 정상 동작해야 했는가?
- 어떤 테스트가 통과해야 했는가?
- 왜 이 시점에서 회귀 테스트를 기대했는가?

```text
예: stack page allocation 구현 이후에도 Project 2 userprog 테스트는 기존처럼 통과해야 했다.
```

### 2.2. 실제 증상

- 실패한 테스트명:
- 실패 유형: 커널 패닉 / FAIL / TIMEOUT / 빌드 실패 / assertion 실패 / 잘못된 출력
- 최초 발견 시점:
- 영향 범위: 특정 테스트 1개 / 여러 userprog 테스트 / 전체 테스트

```text
예: tests/userprog/create-null이 FAIL 처리되었고, initd에서 kernel panic이 발생했다.
```

## 3. 재현 방법

실제로 실행한 명령을 그대로 남긴다. 나중에 같은 커밋으로 checkout해서 다시 확인할 수 있어야 한다.

```bash
git switch --detach <문제_발생_commit>
cd pintos/vm
make clean
make <테스트_타깃_또는_check_명령>
```

단일 테스트를 직접 실행한 경우:

```bash
cd pintos/vm/build
make tests/userprog/<test-name>.result
```

VM 빌드 기준으로 Project 2 테스트 묶음을 실행한 경우:

```bash
cd pintos/vm
make clean
make TEST_SUBDIRS="tests/userprog tests/filesys/base tests/userprog/no-vm tests/threads" \
     GRADING_FILE='$(SRCDIR)/tests/userprog/Grading.no-extra' \
     check
```

## 4. 실패 로그

핵심 로그만 붙인다. 전체 로그가 길면 파일 경로를 함께 남긴다.

### 4.1. 요약 결과

```text
FAIL tests/userprog/<test-name>
```

### 4.2. 핵심 에러 로그

```text
<kernel panic, assertion, diff, timeout 등 핵심 로그>
```

### 4.3. 로그 파일 위치

| 파일 | 용도 |
| --- | --- |
| `pintos/vm/build/results` | 전체 테스트 pass/fail 요약 |
| `pintos/vm/build/tests/.../<test>.output` | 테스트 실행 출력 |
| `pintos/vm/build/tests/.../<test>.errors` | stderr, panic, debug 출력 |
| `pintos/vm/build/tests/.../<test>.result` | 채점 결과 |

## 6. 원인

### 6.1. 직접 원인

```text
예: setup_stack()에서 page allocation과 claim은 성공했지만, success 변수를 true로 바꾸지 않아 성공 경로에서도 false를 반환했다.
```

### 6.2. 왜 테스트 실패로 이어졌는가?

```text
예: setup_stack() false 반환 → load 실패 → process_exec() -1 반환 → initd()가 launch 실패로 판단 → kernel panic.
```
