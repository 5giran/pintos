## 질문 목록
Q.Stack growth란?
A.Stack growth는 stack이 더 필요할 때 page fault를 계기로 새 anonymous page를 추가해서 stack 영역을 확장하는 기능이다.

Q. bit map 이란?
A. bitmap은 말 그대로 bit들의 배열입니다.

Q. dirty bit는 문자열로 예를 들면 매핑 되어 있는 데이터에서 어느 부분이 수정이 되었는지 알 수 있는가? 아니면, 단지 그 주소에 있는 값이 변경 되었는지 여부만 판단하는가?
A. 해당 페이지가 변경된 상태인 것을 알려주기는 하지만, 정확히 어느 부분이 변경되었는지 byte로는 알려주지 않는다.

Q. Memory-mapped File 에서 lazy loading이나, dirty bit 여부는 이해가 가지만,
eviction은 왜 필요한가? 할당 후 접근할 때 디스크에 가져오는게 lazy loading 구현이고, 수정 했을 때 수정 여부를 체크 하는게 dirty bit 맵이며 수정을 했다면 원본에도 munmap으로 수정한 후에 그냥 eviction때 그냥 빼면 되는거 아닌가?
