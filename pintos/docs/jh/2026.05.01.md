## 오늘 할 일

### Argument Passing 구현

현재 Pintos는 프로그램 이름만 전달하면 실행할 수 있는 상태이다.

예를 들어 아래처럼 프로그램 이름만 넘기면 실행된다.

```c
process_exec("grep");
```

하지만 아래처럼 프로그램 이름 뒤에 인자를 함께 넘겼을 때는, 문자열을 파싱해서 사용자 프로그램에 인자로 전달해야 한다.

```c
process_exec("grep foo bar");
```

최종적으로 사용자 프로그램에서는 다음과 같은 형태로 받을 수 있어야 한다.

```c
argc == 3

argv[0] == "grep"
argv[1] == "foo"
argv[2] == "bar"
argv[3] == NULL
```

즉, 내가 구현해야 하는 것은 다음과 같다.

> 명령어 문자열을 공백 기준으로 나누고,  
> `argc`, `argv` 형태로 만들어서  
> 사용자 프로그램의 `_start(argc, argv)`에 전달되도록  
> 사용자 스택과 레지스터를 세팅하는 것.

---

## 사용자 프로그램 시작 흐름

Pintos에서 사용자 프로그램은 `main()`에서 바로 시작하지 않는다.

먼저 `_start()` 함수에서 시작한다.

```c
void
_start (int argc, char *argv[]) {
    exit (main (argc, argv));
}
```

실행 흐름은 다음과 같다.

```text
커널
  ↓
_start(argc, argv)
  ↓
main(argc, argv)
  ↓
exit()
```

따라서 커널은 사용자 프로그램을 시작하기 전에 `_start()`가 받을 인자를 미리 준비해야 한다.

---

## x86-64 인자 전달 규칙

64비트 x86-64 환경에서는 함수 인자를 전달할 때, 처음 몇 개의 인자는 스택이 아니라 레지스터에 넣는다.

`_start()`의 함수 형태는 다음과 같다.

```c
_start(int argc, char *argv[])
```

`_start()`에 전달해야 하는 인자는 두 개이다.

| 인자 | 의미 | 전달 위치 |
|---|---|---|
| `argc` | argument count, 인자의 개수 | `RDI` |
| `argv` | argument vector, 문자열 포인터 배열 | `RSI` |

즉, 사용자 프로그램 시작 직전에는 다음과 같이 세팅되어야 한다.

```text
RDI = argc
RSI = argv
```

예를 들어 다음 명령어가 들어왔다고 하자.

```text
/bin/ls -l foo bar
```

공백 기준으로 나누면 다음과 같다.

```text
argv[0] = "/bin/ls"
argv[1] = "-l"
argv[2] = "foo"
argv[3] = "bar"
argv[4] = NULL

argc = 4
```

따라서 레지스터는 다음과 같이 설정되어야 한다.

```text
RDI = 4
RSI = argv 배열의 시작 주소
```

여기서 중요한 점은 `RSI`에 `"/bin/ls"` 문자열의 주소를 넣는 것이 아니라, `argv[0]` 포인터가 저장된 배열의 시작 주소를 넣어야 한다는 것이다.

---

## argc와 argv의 차이

`argc`는 단순한 숫자이다.

```c
argc = 4;
```

따라서 `RDI` 레지스터에 바로 넣으면 된다.

하지만 `argv`는 문자열 포인터들의 배열이다.

```c
char *argv[]
```

즉, 구조는 다음과 같다.

```text
argv
 ├── argv[0] → "/bin/ls"
 ├── argv[1] → "-l"
 ├── argv[2] → "foo"
 ├── argv[3] → "bar"
 └── argv[4] → NULL
```

사용자 프로그램이 이 값들에 접근할 수 있어야 하므로, 문자열과 포인터 배열을 사용자 스택에 직접 저장해야 한다.

---

## 스택의 특징

스택은 높은 주소에서 낮은 주소 방향으로 자란다.

즉, 처음 스택 주소가 높은 곳에 있다면, 데이터를 push할 때마다 스택 포인터의 주소는 감소한다.

```text
높은 주소
0x47480000
    ↓ push
0x4747fff8
    ↓ push
0x4747fff0
낮은 주소
```

---

## 스택에 들어가는 데이터

Argument Passing을 구현할 때 스택에는 크게 세 종류의 데이터가 들어간다.

### 1. 실제 문자열

각 인자 문자열을 스택에 저장한다.

```text
"/bin/ls\0"
"-l\0"
"foo\0"
"bar\0"
```

문자열 끝에는 반드시 널 문자 `\0`이 들어가야 한다.

예를 들어 `"foo"`는 실제 메모리에 다음과 같이 저장된다.

```text
'f' 'o' 'o' '\0'
```

---

### 2. 문자열을 가리키는 포인터 배열

문자열을 스택에 저장한 뒤, 각 문자열의 시작 주소를 따로 저장한다.

이 주소들이 `argv` 배열의 요소가 된다.

```text
argv[0] = "/bin/ls" 문자열의 시작 주소
argv[1] = "-l" 문자열의 시작 주소
argv[2] = "foo" 문자열의 시작 주소
argv[3] = "bar" 문자열의 시작 주소
argv[4] = NULL
```

`argv[argc]`는 반드시 `NULL`이어야 한다.

```c
argv[argc] == NULL
```

---

### 3. Fake Return Address

마지막으로 fake return address를 스택에 넣는다.

일반적인 함수 호출에서는 `call` 명령어가 return address를 스택에 자동으로 넣는다.

하지만 `_start()`는 커널이 직접 시작시키는 함수이기 때문에 일반적인 `call` 명령으로 호출된 것이 아니다.

그래도 스택 구조를 일반 함수 호출과 비슷하게 맞추기 위해 가짜 반환 주소를 넣는다.

```text
fake return address = 0
```

---

## 최종 스택 상태 예시

다음 명령어를 실행한다고 가정한다.

```text
/bin/ls -l foo bar
```

최종 스택 상태는 대략 다음과 같다.

| 주소 | 내용 |
|---|---|
| `0x4747fffc` | `"bar\0"` |
| `0x4747fff8` | `"foo\0"` |
| `0x4747fff5` | `"-l\0"` |
| `0x4747ffed` | `"/bin/ls\0"` |

| `0x4747ffe8` | 정렬용 padding |
| `0x4747ffe0` | `argv[4] = NULL` |
| `0x4747ffd8` | `argv[3] = "bar"` 주소 |
| `0x4747ffd0` | `argv[2] = "foo"` 주소 |
| `0x4747ffc8` | `argv[1] = "-l"` 주소 |
| `0x4747ffc0` | `argv[0] = "/bin/ls"` 주소 |
| `0x4747ffb8` | fake return address |

이때 레지스터는 다음과 같이 설정된다.

```text
RDI = 4
RSI = 0x4747ffc0
```

`0x4747ffc0`은 `"/bin/ls"` 문자열의 주소가 아니다.

이 주소는 `argv[0]` 포인터가 저장된 위치이다.

즉, `argv` 배열의 시작 주소이다.

---

## 구현 순서

### 1. 명령어 문자열 복사

입력으로 받은 명령어 문자열을 바로 수정하면 위험할 수 있으므로, 보통 복사본을 만든다.

예를 들어 다음 문자열이 들어왔다고 하자.

```c
"grep foo bar"
```

이 문자열을 복사한 뒤 파싱에 사용한다.

---

### 2. 공백 기준으로 파싱

명령어 문자열을 공백 기준으로 나눈다.

```text
"grep foo bar"
```

파싱 결과는 다음과 같아야 한다.

```text
argv[0] = "grep"
argv[1] = "foo"
argv[2] = "bar"
argv[3] = NULL

argc = 3
```

여러 개의 공백은 하나의 공백처럼 처리해야 한다.

```text
"grep   foo    bar"
```

위 문자열도 다음과 동일하게 처리되어야 한다.

```text
"grep foo bar"
```

Pintos에서는 `strtok_r()` 사용을 권장한다.

```c
token = strtok_r(command, " ", &save_ptr);
```

필요하다면 공백뿐 아니라 탭, 개행 문자까지 구분자로 처리할 수 있다.

```c
token = strtok_r(command, " \t\n", &save_ptr);
```

---

### 3. 프로그램 이름만 사용해서 실행 파일 열기

실행 파일을 열 때는 전체 명령어 문자열을 넘기면 안 된다.

잘못된 예시는 다음과 같다.

```c
load("grep foo bar", ...);
```

이렇게 하면 `"grep foo bar"`라는 이름의 파일을 찾으려고 한다.

실제로 실행할 파일 이름은 첫 번째 토큰인 `argv[0]`이다.

올바른 방식은 다음과 같다.

```c
load("grep", ...);
```

즉, `load()` 또는 실행 파일을 여는 부분에는 반드시 `argv[0]`만 넘겨야 한다.

---

### 4. 문자열들을 사용자 스택에 복사

파싱한 문자열들을 사용자 스택에 복사한다.

보통 뒤에서부터 넣는다.

```text
bar
foo
-l
/bin/ls
```

각 문자열을 스택에 복사한 뒤에는 해당 문자열이 저장된 시작 주소를 따로 기억해야 한다.

이 주소들은 나중에 `argv` 배열을 만들 때 사용된다.

---

### 5. 스택 정렬

문서에서는 성능과 ABI 규칙을 위해 스택 포인터를 8의 배수로 맞추라고 한다.

즉, 필요한 경우 padding을 넣어서 스택 포인터를 정렬한다.

```text
정렬 전: 0x4747ffed
정렬 후: 0x4747ffe8
```

---

### 6. `argv[argc] = NULL` push

C 표준에 맞게 `argv` 배열의 마지막에는 반드시 `NULL`을 넣어야 한다.

```c
argv[argc] = NULL;
```

스택에는 널 포인터 값을 push한다.

```text
0
```

---

### 7. argv 포인터들을 오른쪽에서 왼쪽으로 push

각 문자열의 시작 주소를 스택에 저장한다.

예를 들어 다음과 같은 인자들이 있다면,

```text
argv[0] = "/bin/ls"
argv[1] = "-l"
argv[2] = "foo"
argv[3] = "bar"
```

포인터는 오른쪽에서 왼쪽 순서로 push한다.

```text
push argv[4] = NULL
push argv[3] = "bar" 주소
push argv[2] = "foo" 주소
push argv[1] = "-l" 주소
push argv[0] = "/bin/ls" 주소
```

이렇게 해야 메모리상에서 `argv[0]`, `argv[1]`, `argv[2]`가 배열처럼 연속적으로 배치된다.

---

### 8. argv 배열의 시작 주소 저장

`argv[0]` 포인터가 저장된 위치가 곧 `argv` 배열의 시작 주소이다.

이 주소를 나중에 `RSI`에 넣어야 한다.

```text
argv_start = &argv[0]
```

---

### 9. Fake Return Address push

마지막으로 fake return address를 push한다.

```text
push 0
```

최종 스택 포인터는 이 fake return address를 가리키게 된다.

---

### 10. 레지스터 설정

사용자 프로그램 시작 직전에 레지스터를 설정한다.

```text
RDI = argc
RSI = argv_start
```

예를 들어 `/bin/ls -l foo bar`라면 다음과 같다.

```text
RDI = 4
RSI = argv 배열의 시작 주소
```

다시 주의할 점은, `RSI`에는 첫 번째 문자열의 주소가 아니라 `argv` 배열의 시작 주소를 넣어야 한다는 것이다.

---

## 최종 정리

Argument Passing 구현의 핵심은 다음 문자열을

```text
"grep foo bar"
```

아래와 같은 형태로 바꾸는 것이다.

```c
argc = 3;

argv[0] = "grep";
argv[1] = "foo";
argv[2] = "bar";
argv[3] = NULL;
```

그리고 사용자 프로그램이 시작할 때 `_start(argc, argv)`가 정상적으로 받을 수 있도록 사용자 스택과 레지스터를 직접 세팅해야 한다.

핵심 체크리스트는 다음과 같다.

- 명령어 문자열을 복사한다.
- 공백 기준으로 파싱한다.
- 실행 파일 이름은 `argv[0]`만 사용한다.
- 문자열들을 사용자 스택에 복사한다.
- 각 문자열의 시작 주소를 저장한다.
- 스택을 8바이트 단위로 정렬한다.
- `argv[argc] = NULL`을 넣는다.
- `argv` 포인터 배열을 스택에 넣는다.
- fake return address를 넣는다.
- `RDI = argc`, `RSI = argv 시작 주소`로 설정한다.
