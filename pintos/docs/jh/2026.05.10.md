## 학습
pintos에서 대부분은 읽어오는 파일은 ANON TYPE이다. 대표적으로 file 같은 경에는 mmap이 사용 된다.

load에서 파일을 읽어오고, load_segment에서 페이지 단위로 메모리에 load한다.
VM은 실제 메모리에 올리는 것이 아니므로 기존에 사용하던 load_segment와는 다른 코드를 작성해야 한다.

load_segment 내부에서 while문으로 페이지 단위로 접근을 하고, 매번 vm_alloc_page_with_initializer를 호출한다. SPT에 적재할 PAGE에 PAGE FAULT가 일어났을 때 실제 디스크에서 메모리에 load를 해야 하므로, load에 필요한 모든 정보를 PAGE에 넘겨 줘야 한다. 따라서 새로운 구조체를 선언하고, 해당 구조체에 file과 offset(어디서부터 읽을지) 그리고 얼만큼 읽을지 적혀있는 read_bytes도 같이 넘겨야 한다. 

SPT에서는 해시 테이블을 사용해야 한다. 
Pintos의 hash table은 체이닝 방식으로 이루어져 있다. 핵심은 해시 테이블이 원소 노드를 직접 할당하지 않고, 사용자가 만든 구조체 안에 struct hash_elem을 넣어둔다.

struct hash 구성 요소
- elem_cnt: 현재 들어 있는 원소 수
- bucket_cnt: 버킷 개수, 항상 2의 거듭제곱
- buckets: struct list 배열
- hash: 원소의 hash 값을 계산하는 콜백
- less: 두 원소를 비교하는 콜백
- aux: hash, less에 같이 넘겨주는 보조 데이터
