## 정리
가상메모리 구현
기존에 가진 메모리 보다, 더 많은 메모리가 있는 것처럼 동작이 가능.
- 프로세스가 실행 중이면, 해당프로세스 동작에 필요한 내용들을 디스크에서 메모리로 올려야 한다. 하지만 가상 메모리는 해당 내용들을 메모리에 올려두지 않고, 어딘가에 메모만 해두고 접근할 때 page fault를 발생시켜 그 때서야 디스크에 접근하여 해당 내용을 가져온다. 또한 물리 메모리 영역이 가득 차있다면 swap을 통해 자주 사용하지 않은 내용을 디스크로 보내고 지금 당당 필요한 내용을 메모리에 올린다. 

Page 
가상 메모리 영역, 한 페이지당 크기는 4KB

Frame
물리 메모리 영역, 한 프레임당 크기는 4KB

Page table (커널 메모리)
페이지와 프레임 매핑 되어 있는 주소.
자료구조는 해시테이블로 되어 있으며, 각 프로세스마다 자체 테이블을 가진다.

Swap slot (디스크)
물리 메모리가 부족할 때, frame 내용을 swap slot에 저장하고 빈 frame을 새로운 가상 주소에 할당한다. swap slot에 저장된 page가 다시 필요해지면, swap slot에서 내용을 읽어 새 frame에 복구한다.

Supplemental Page Table SPT
각 가상 페이지가 현재 어떤 상태이고, page fault가 나면 어떻게 처리해야 하는지 저장하는 테이블
